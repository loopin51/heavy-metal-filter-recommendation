# 중금속 흡착 필터 추천 알고리즘 설명

> 대상 코드: [`backend/engine/recommender.py`](../backend/engine/recommender.py)
> 설계 근거: [`model_design.md`](./model_design.md)
> 버전: 1.0 · 2026-06

이 문서는 추천 엔진이 **입력(오염 조건) → 출력(필터 구조 + 제거율 추정)** 을 어떻게
계산하는지 설명하고, 마지막에 실제 입력 하나를 숫자까지 따라가며 검증합니다.

---

## 0. 큰 그림: 왜 규칙 기반 + 스코어링 하이브리드인가

순수 머신러닝(ML)이 아니라 **결정론적 규칙 엔진 + 데이터 기반 스코어링**을 결합했습니다.

| 기준 | 순수 ML | 규칙 + 스코어링 (채택) |
|------|---------|------------------------|
| 학습 데이터 | 수천~수만 건 필요 | DB 55 레코드로 동작 |
| 설명 가능성 | 낮음 (블랙박스) | 높음 (각 추천에 근거 제시) |
| 과학적 신뢰성 | 학습 편향 위험 | 문헌 메커니즘 직접 반영 |
| 업데이트 | 재학습 필요 | DB 행 추가로 즉시 반영 |

- **Phase 2(규칙 엔진)** 는 결정론적입니다 — 같은 입력이면 항상 같은 적층 구조.
- **Phase 3(스코어링)** 는 데이터 주도적입니다 — DB 값으로 제거율을 추정.

입력은 4가지입니다.

| 필드 | 값 | 의미 |
|------|----|----|
| `scenario` | industrial / mining / agricultural / groundwater / urban | 오염 시나리오 |
| `metals` | Pb²⁺, Cu²⁺, … (1개 이상) | 대상 중금속 |
| `level` | low / medium / high | 오염도 (<5 / 5–100 / >100 mg/L) |
| `pH_range` | acidic / neutral / alkaline | 유입수 pH (<5 / 5–7 / >8) |

이 입력이 3단계 파이프라인을 통과합니다.

```
[입력] → Phase 1 전처리 → Phase 2 규칙 엔진 → Phase 3 스코어링 → [추천 결과]
```

---

## 1. Phase 1 — 전처리: 금속을 전하로 분류

모든 흡착 전략의 출발점은 **중금속을 양이온 / 음이온으로 나누는 것**입니다.

```python
CATION_METALS = {"Pb2+", "Cu2+", "Cd2+", "Cr3+", "Hg2+", "Zn2+", "Ni2+"}
ANION_METALS  = {"Cr6+", "As3+", "As5+"}
```

양이온과 음이온은 제거 메커니즘이 **근본적으로 다르기** 때문입니다.

- **양이온** → 제올라이트(이온교환), 키토산(킬레이트 착화)
- **음이온** → 철산화물(≡FeOH 표면 착화)

이 분류 결과가 Phase 2에서 "철산화물층(슬롯 4)을 켤지"를 결정합니다.

---

## 2. Phase 2 — 규칙 엔진: 9칸 슬롯 켜고 끄기 (결정론적)

물이 위(유입)에서 아래(처리수)로 흐르는 순서대로 **9개의 고정 슬롯**이 있습니다.
각 슬롯은 초기 `active`(켜짐) 상태를 가지며, 4개는 항상 켜진 **필수층**입니다.

| 슬롯 | 소재 | 초기 상태 | 역할 |
|:--:|------|:--:|------|
| 1 | 탄산칼슘 (CaCO₃) | ⚫ 꺼짐 | pH 완충 (산성 중화) |
| 2 | 이산화망간 (MnO₂) | ⚫ 꺼짐 | As(III) → As(V) 산화 |
| 3 | 규사 | 🟢 **필수** | 물리적 전처리 여과 |
| 4 | 철산화물 | ⚫ 꺼짐 | 음이온(As·Cr⁶⁺) 흡착 |
| 5 | 천연 제올라이트 | 🟢 **필수** | 양이온 이온교환 주흡착 |
| 6 | 활성탄 | 🟢 **필수** | 광역 흡착 (중금속+유기물) |
| 7 | 키토산 | ⚫ 꺼짐 | 킬레이트 고용량 흡착 |
| 8 | 변성 바이오차 | ⚫ 꺼짐 | 고부하 다기능 복합 |
| 9 | 자갈 | 🟢 **필수** | 지지·배수 |

여기에 **3종류의 규칙을 순서대로** 적용합니다.

### 2-1. pH 규칙 (`apply_pH_rules`)

- **산성 (pH < 5)** → 슬롯 1(탄산칼슘) 켜짐. 산을 먼저 중화해야 뒤쪽 이온교환·킬레이트 소재가
  제 성능을 냅니다.
- **염기성 (pH > 8)** → `ALKALINE_PRECIPITATION` 경고. 금속이 수산화물 M(OH)ₙ로 침전해 필터를 막음.

### 2-2. 오염도 규칙 (`apply_level_rules`)

- **보통 또는 높음** → 슬롯 7(키토산) 켜짐
- **높음** → 슬롯 8(바이오차)도 켜짐

### 2-3. 금속 특화 규칙 (`apply_metal_rules`)

- 음이온(As·Cr⁶⁺)이 있으면 → 슬롯 4(철산화물) 켜짐
- **As³⁺ 가 있으면** → 슬롯 2(MnO₂)도 켜짐 + 경고. As(III)는 중성 분자(H₃AsO₃)라 흡착이
  어렵습니다. **MnO₂로 먼저 As(V)로 산화**시킨 뒤 철산화물이 잡는 순서가 필수이므로,
  슬롯 2(MnO₂)가 슬롯 4(철산화물)보다 앞에 배치됩니다.
- Hg가 있으면 → `HG_ENHANCED_AC` (황 함침 활성탄 권고, info)
- Cr⁶⁺ + 양이온 공존 → `CR6_PH_CONFLICT` (Cr⁶⁺는 산성에서, 양이온은 중성에서 잘 잡혀 pH 충돌)

> 충돌 해결(`model_design.md` §4.4): **높음 + 염기성** 이면 `HIGH_ALKALINE_LOAD`(danger) —
> 흡착 필터보다 화학적 침전 공정이 우선이라는 강한 경고를 추가합니다.

### 2-4. 적층 구조 확정

마지막에 **켜진 슬롯만 슬롯 번호 순서대로** 추출합니다.

```python
active = [s for s in template if s["is_active"]]   # 슬롯 순서 보존
```

슬롯 번호 = 물 흐름 순서이므로, **자갈(9)은 항상 맨 아래, 탄산칼슘(1)은 켜지면 항상 맨 위**가
됩니다. 켜진 레이어 수는 입력에 따라 최소 4개 ~ 최대 9개로 가변적입니다.

## 3. Phase 3 — 스코어링: 제거율을 DB에서 계산

핵심 단계입니다. 켜진 각 레이어가 각 금속을 얼마나 잡는지를 **`adsorption_data.json` 레코드에서
계산**합니다. (수치를 임의로 만들거나 하드코딩하지 않습니다.)

### 3-1. 한 레이어의 단일 금속 제거율 (`estimate_removal_per_layer`)

(소재 × 금속) 레코드를 찾되, **데이터 품질이 가장 높은 것**을 고릅니다
(우선순위 `high > medium > low > estimated`). 그리고 문헌 실험값에 3가지 보정을 곱합니다.

```
조정값(adjusted) = (문헌 removal_pct / 100) × 컬럼보정 × pH보정
```

| 보정 | 값 | 이유 |
|------|----|----|
| **컬럼 보정** (`COLUMN_FACTOR`) | 배치 데이터 ×0.40, 컬럼 데이터 ×1.0 | 문헌값은 대부분 실험실 *배치(비커)* 조건이라 과대평가됨. 실제 *컬럼(필터)* 효율은 평균 40% 수준 (PMC10890072) |
| **pH 보정** (`get_ph_factor`) | 최적 범위 내 1.0 / 경계 ±1 0.6 / 벗어남 0.2 | 입력 pH가 그 소재 최적 pH와 맞는지 |

여기에 레이어 간 공통으로 **경쟁 흡착 페널티**가 곱해집니다.

```
경쟁 페널티 = 1 / (1 + 0.1 × 금속 수)      # 금속 2종 → 0.83, 3종 → 0.77
R_i(금속) = adjusted × 경쟁 페널티
```

복수 금속이 공존하면 서로 흡착 자리를 다퉈 효율이 떨어지는 것을 반영합니다.

### 3-2. 여러 레이어의 누적 제거율 (`compute_cumulative_removal`)

레이어들이 직렬로 작동한다고 보고, **"통과해서 살아남는 비율"을 곱해 나갑니다.**

```
R_total = 1 − Π (1 − R_i)
```

예) 한 층이 60%, 다음 층이 50%를 잡으면 → 통과율 0.4 × 0.5 = 0.2 → **총 제거율 80%**.
한 층만으로 부족해도 여러 층을 거치며 누적되는 구조입니다. (100% 방지를 위해 각 R_i는 0.999로 클램프)

### 3-3. 설명 가능성을 위해 함께 기록하는 값

| 필드 | 의미 |
|------|----|
| `supporting_records` | 계산에 실제 사용한 레코드 ID (예: `ADS-001`) |
| `limiting_layer` | 기여도가 가장 낮은 **병목 레이어** |
| `confidence` | 사용 레코드 중 **최저 품질** 기준 (low면 경고) |
| `correction_applied` | 적용된 보정 전체 ("배치→컬럼 ×0.4, 경쟁흡착 ×0.83 …") |

## 4. 마지막 조립 (`recommend`)

위 결과를 모아 다음을 수행합니다.

1. `materials.json`에서 소재 상세(반응식·두께·최적 pH·재생법)를 붙여 `FilterLayer` 구성
2. 두께 계산: `high` → `high_load` 값, 그 외 → `(min+max)/2`
3. 신뢰도 집계(최저 등급 기준) + `LOW_CONFIDENCE` 경고(해당 시)
4. **항상** `COLUMN_CORRECTION`(컬럼 보정 고지)과 면책 문구 추가

> **한 줄 요약**: 규칙으로 *"어떤 층을 쌓을지"* 를 결정(Phase 2)하고, DB 데이터로
> *"얼마나 잡을지"* 를 보수적으로 추정(Phase 3)하되, 모든 수치에 근거 레코드와 보정 계수를
> 붙여 검증 가능하게 만든다.

## 5. 실제 동작 예시 (숫자까지 추적)

아래 값은 실행 중인 엔진에서 직접 추출한 **실제 계산값**입니다.

### 입력

```json
{ "scenario": "mining", "metals": ["Pb2+", "As3+"], "level": "high", "pH_range": "acidic" }
```

### Phase 1 — 전처리

```
양이온(cations) = { Pb2+ }
음이온(anions)  = { As3+ }
```

### Phase 2 — 규칙 엔진

적용되는 규칙:
- pH = acidic → 슬롯 1(탄산칼슘) ON, 경고 없음
- level = high → 슬롯 7(키토산) ON, 슬롯 8(바이오차) ON
- anions={As3+} → 슬롯 4(철산화물) ON
- As3+ 존재 → 슬롯 2(MnO₂) ON + `AS3_OXIDATION_REQUIRED` 경고

결과 적층 구조 (9층 전부 활성):

```
유입수 ↓
 1  탄산칼슘        (조건부 · 산성 중화)
 2  이산화망간 MnO₂ (조건부 · As(III) 산화 전처리)
 3  규사            (필수)
 4  철산화물        (조건부 · As3+ 음이온 제거)
 5  천연 제올라이트  (필수)
 6  활성탄          (필수)
 7  키토산          (조건부 · 오염도 보통↑)
 8  변성 바이오차    (조건부 · 오염도 높음)
 9  자갈            (필수)
처리수 ↓
```

### Phase 3 — 스코어링

금속 2종이므로 **경쟁 페널티 = 1/(1+0.1×2) = 0.833**.

#### Pb²⁺ 레이어별 기여도 (pH=acidic)

| 슬롯 | 소재 | 레코드 | 문헌 제거율 | 조건 | 컬럼보정 | pH보정 | **R_i** |
|:--:|------|------|:--:|:--:|:--:|:--:|:--:|
| 1 | 탄산칼슘 | ADS-033 | 75% | batch | 0.40 | 1.0 | 25.0% |
| 2 | MnO₂ | ADS-038 | 90% | batch | 0.40 | 0.6 | **18.0%** ← 병목 |
| 3 | 규사 | — | (데이터 없음, 건너뜀) | | | | — |
| 4 | 철산화물 | ADS-032 | 92% | batch | 0.40 | 1.0 | 30.7% |
| 5 | 제올라이트 | ADS-008 | 98% | batch | 0.40 | 1.0 | 32.7% |
| 6 | 활성탄 | ADS-016 | 95% | batch | 0.40 | 1.0 | 31.7% |
| 7 | 키토산 | ADS-001 | 98% | batch | 0.40 | 1.0 | 32.7% |
| 8 | 바이오차 | ADS-026 | 94% | **column** | **1.0** | 1.0 | 78.3% |
| 9 | 자갈 | — | (데이터 없음, 건너뜀) | | | | — |

- R_i 한 줄 예시(슬롯 2): `0.90 × 0.40(배치) × 0.6(pH경계) × 0.833(경쟁) = 0.180 → 18.0%`
- 슬롯 8(바이오차)은 **컬럼 데이터**라 보정이 ×1.0 → 가장 높은 기여(78.3%)
- 누적: `R_total = 1 − (1−0.25)(1−0.18)(1−0.307)(1−0.327)(1−0.317)(1−0.327)(1−0.783)`
  **→ 97.1%**

#### As³⁺ 레이어별 기여도 (pH=acidic)

| 슬롯 | 소재 | 레코드 | 문헌 제거율 | 품질 | 컬럼보정 | pH보정 | **R_i** |
|:--:|------|------|:--:|:--:|:--:|:--:|:--:|
| 2 | MnO₂ | ADS-037 | 95% | medium | 0.40 | 1.0 | 31.7% |
| 4 | 철산화물 | ADS-030 | 65% | medium | 0.40 | 1.0 | **21.7%** ← 병목 |

> 나머지 레이어(탄산칼슘·규사·제올라이트·활성탄·키토산·바이오차·자갈)는 As³⁺에 대한 흡착
> 레코드가 없어 **계산에서 제외**됩니다. As(III)는 본래 제거가 어려운 종임을 그대로 보여줍니다.

- 누적: `R_total = 1 − (1−0.317)(1−0.217)` **→ 46.5%**

### 최종 출력 (요약)

```
filter_stack : 9층 (위 구조)
warnings     : [caution] AS3_OXIDATION_REQUIRED
               [info]    COLUMN_CORRECTION
               [info]    COMPETITIVE_PENALTY
metal_removal:
  Pb²⁺ : 97.1%  신뢰도 medium  병목=MnO₂      근거 7개 레코드
  As³⁺ : 46.5%  신뢰도 medium  병목=철산화물   근거 2개 레코드 (ADS-037, ADS-030)
confidence   : medium (0.65)
design_params: 총 두께 105–165 cm · 9층 · HLR 2.0 m/h · EBCT 18.5분
```

### 이 결과가 말해주는 것

- **Pb²⁺ 97.1%** : 양이온이라 제올라이트·키토산·바이오차 등 여러 층이 협력 → 높은 누적 제거율.
- **As³⁺ 46.5%** : 음이온 중 가장 까다로운 종. 2개 층만 기여하고, 더 잘 잡는 철산화물이
  pH(산성)·배치 보정으로 깎여 **병목**이 됨. 알고리즘이 "As(III)는 산화 전처리가 필수"라는
  현실(경고 + 낮은 추정치)을 그대로 드러냅니다.
- 모든 수치는 **근거 레코드 ID와 보정 계수**가 붙어 있어, 사용자가 "왜 이 값인가"를 추적할 수
  있습니다.

## 6. 모델의 한계 (항상 명시)

- 모든 제거율은 배치 조건 문헌값에 **컬럼 보정(×0.40)** 을 적용한 **이론적 추정값**입니다.
- 현장 실제 성능은 이론값의 **50–80%** 수준으로 보수적으로 기대해야 합니다.
- `column_factor`는 소재 무관 고정값, 경쟁 흡착은 단순 페널티, 온도·장기 파울링은 미반영.
- **실제 시설 설계 전 반드시 파일럿 컬럼 실험과 전문가 검토가 필요합니다.**
