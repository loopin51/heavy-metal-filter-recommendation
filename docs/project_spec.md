## 1. 프로젝트 개요

### 1.1 목적

오염수의 조건(오염 유형, 중금속 종류, 오염도 수준, pH 범위)을 입력받아
**다층 흡착 필터 구조**를 추천하고, 각 필터 소재의 흡착 원리와 문헌 근거를 함께 제시하는 웹 서비스.

### 1.2 배경 및 필요성

- 중금속 수처리 분야에서 "어떤 소재를 어떤 순서로 쌓아야 하는가"에 대한
  일반인·연구자 접근 가능한 통합 가이드 도구가 부재
- 관련 지식이 수백 편의 분산된 학술 논문에 산재
- 본 서비스는 동료심사 문헌에서 추출·정규화한 DB(`/db/` 폴더)와
  규칙 기반 추천 모델(`model_design.md`)을 기반으로 이 공백을 메움

### 1.3 핵심 원칙

- **근거 기반**: 모든 추천은 `adsorption_data.json` 내 실험 문헌 레코드에 연결
- **투명한 추론**: 추천 결과에 반응식·메커니즘·인용 출처를 함께 출력
- **보수적 추정**: 제거율 추정치는 배치→컬럼 보정(×0.40) 후 표시
- **단순한 입력**: 4가지 조건만으로 추천 생성 (전문 지식 불필요)

### 1.4 프로젝트 범위 (Scope)

| 포함 | 제외 |
|------|------|
| 필터 층 구성 추천 | 필터 시공·설치 가이드 |
| 소재별 흡착 원리 설명 | 실시간 수질 측정·IoT 연동 |
| 문헌 기반 성능 수치 제공 | 상용 제품 구매 링크 |
| 경고·주의사항 출력 | 법적 규제 준수 판단 |
| 소재 사전(카탈로그) | 설계 도면 자동 생성 |

### 2.2 F-01: 입력 폼 상세

```
조건 1. 오염 상황 유형 (단일 선택)
  - 산업 폐수 (도금·염색·제련)
  - 광산 배수 AMD
  - 농업용수 오염
  - 지하수 오염
  - 도시 비점오염

조건 2. 중금속 종류 (복수 선택, 최소 1개)
  - Pb²⁺ 납 / Cu²⁺ 구리 / Cd²⁺ 카드뮴 / Cr³⁺/⁶⁺ 크롬
  - Hg²⁺ 수은 / As³⁺/⁵⁺ 비소 / Zn²⁺ 아연 / Ni²⁺ 니켈

조건 3. 오염도 수준 (단일 선택)
  - 낮음 (< 5 mg/L)     → 기본 필터
  - 보통 (5–100 mg/L)   → 기본 + 키토산
  - 높음 (> 100 mg/L)   → 기본 + 키토산 + 바이오차

조건 4. pH 범위 (단일 선택)
  - 산성 (pH < 5)       → CaCO₃ 완충층 맨 앞 삽입
  - 약산성~중성 (pH 5–7) → 기본형 유지
  - 염기성 (pH > 8)     → 경고 문구 출력
```

**유효성 검사 규칙**

- 중금속 미선택 상태에서 분석 버튼 비활성화
- 오염 유형·오염도·pH 중 하나라도 미선택 시 분석 버튼 비활성화

### 2.3 F-02 ~ F-07: 출력 명세

**레이어 카드 필수 포함 정보**

```
각 레이어마다:
  - 슬롯 번호 (1–9)
  - 소재 이름 (한국어 + 영문)
  - 역할 한 줄 요약
  - 핵심 흡착 메커니즘
  - 반응식 (화학식)
  - 추천 두께 범위 (cm)
  - 적합 중금속 목록
  - 조건부/필수 여부 표시

경고 카드:
  - severity 아이콘 (info / caution / danger)
  - 경고 코드
  - 설명 문구
  - 권고 조치
```

**pH 염기성 경고 문구 (F-05) — 고정 텍스트**

```
⚠️ 염기성 조건 주의 — 금속 수산화물 침전 가능성

pH 8 이상에서는 Fe, Mn, Al 등이 M(OH)ₙ 형태로 침전되어
흡착 필터를 급격히 막을 수 있습니다.

권고 조치:
  · 필터 투입 전 응집–침전 전처리 공정 적용 강력 권장
  · 필터 교체 주기를 일반 조건 대비 30–50% 단축 운영
  · Pb, Cu, Zn, Cd는 pH 8–9에서 수산화물 침전이 지배적
    → 흡착 필터보다 화학 침전 공정이 더 효과적인 구간
```

## 4. 데이터 명세

### 4.1 데이터 파일 구조

모든 데이터는 `/db/` 폴더에 JSON 파일로 관리된다.

```
db/
├── README.md              데이터 출처·신뢰도 등급·참고문헌 목록
├── materials.json         흡착 소재 12종 물성 및 운전 특성
├── heavy_metals.json      중금속 10종 화학 특성 + 국내외 기준치
├── adsorption_data.json   소재 × 중금속 흡착 실험 데이터 (55 레코드)
├── adsorption_data.csv    위와 동일 (분석·시각화용 CSV)
├── filter_rules.json      조건별 필터 구성 규칙
└── scenarios.json         오염 상황 유형 5종
```

### 4.2 데이터 커버리지 현황

| 항목 | 현황 |
|------|------|
| 흡착 소재 | 12종 (자갈·규사·탄산칼슘·MnO₂·철산화물·천연제올라이트·합성4A제올라이트·활성탄·키토산·바이오차·변성바이오차·하이드록시아파타이트) |
| 대상 중금속 | 10종 (Pb·Cu·Cd·Cr³⁺·Cr⁶⁺·Hg·As³⁺·As⁵⁺·Zn·Ni) |
| 흡착 실험 레코드 | 55건 (high 21건, medium 28건, low 6건) |
| 오염 시나리오 | 5종 |
| 데이터 출처 | PubMed/PMC, ScienceDirect, ACS Journals 수록 동료심사 논문 14편 |

### 4.3 데이터 신뢰도 등급

| 등급 | 의미 | 레코드 수 |
|------|------|-----------|
| `high` | 복수 독립 연구, 인용 50+ | 21건 |
| `medium` | 1–2편, 실험 조건 명시 | 28건 |
| `low` | 단일 연구 또는 간접 추정 | 6건 |

### 4.4 데이터 업데이트 정책

- DB는 버전 관리(git tag)로 관리
- 신규 문헌 추가 시 `adsorption_data.json`에 레코드 append
- 레코드 삭제·수정은 히스토리 보존을 위해 `deprecated: true` 플래그 사용
- 사용자에게 데이터 최종 업데이트 날짜 표시 의무

## 5. 추천 모델 명세

> 전체 상세 설계는 `model_design.md` 참조.
> 본 절은 서비스 구현에 필요한 요약만 기술한다.

### 5.1 모델 유형

**하이브리드: 규칙 기반(Rule-based) + 스코어링(Scoring)**

- 규칙 엔진: 결정론적 (동일 입력 → 항상 동일 슬롯 구성)
- 스코어링: 데이터 주도적 (DB 조회 기반 성능 추정)

### 5.2 3단계 처리 파이프라인

```
입력(FilterInput)
    │
    ▼
[Phase 1] 전처리
  - 입력 유효성 검사
  - 금속 전하 분류: cations / anions
  - 오염도·시나리오 정규화
    │
    ▼
[Phase 2] 규칙 엔진  ← filter_rules.json
  - 슬롯 템플릿 초기화 (9슬롯)
  - pH 규칙: acidic → slot 1(CaCO₃) 활성화
             alkaline → 경고 추가
  - 오염도 규칙: medium → slot 7(키토산) / high → slot 7+8(+바이오차) 활성화
  - 금속 특화: anions → slot 4(철산화물) / As(III) → slot 2(MnO₂) 활성화
  - 활성 슬롯 정렬 → FilterStack
    │
    ▼
[Phase 3] 스코어링  ← adsorption_data.json, materials.json
  - 소재 점수 S() → 후보 소재 최적화
  - 누적 제거율 R_total() = 1 − Π(1 − R_i × 0.40 × pH_factor × competition_factor)
  - 신뢰도 점수 C() → confidence 배지
    │
    ▼
출력(FilterRecommendation)
```

### 5.3 오염도별 슬롯 구성 (핵심 규칙)

| 오염도 | 필수 활성 슬롯 | 구성 소재 |
|--------|---------------|-----------|
| 낮음 | 3, 5, 6, 9 | 규사 → 제올라이트 → 활성탄 → 자갈 |
| 보통 | 3, 5, 6, 7, 9 | 위 + 키토산 |
| 높음 | 3, 5, 6, 7, 8, 9 | 위 + 바이오차 |

*pH=산성 시 모든 경우에 slot 1(CaCO₃) 최전방 추가*

### 5.4 성능 추정 보정 계수

| 보정 | 계수 | 근거 |
|------|------|------|
| 배치→컬럼 보정 | × 0.40 | 실험실 qmax 대비 현장 컬럼 효율 평균 |
| 경쟁 흡착 페널티 | ÷ (1 + 0.1 × n_metals) | 복수 금속 공존 시 선택성 감소 |
| pH 비호환 | × 0.2–1.0 | 소재별 최적 pH 일치 여부 |

**중요**: 모든 추정 수치는 이론값이며 현장 성능의 50–80% 수준으로 보수적으로 해석해야 함.
결과 화면에 이 고지사항을 항상 표시한다.

```

### 6.2 데이터 흐름

```
사용자 입력
    │  POST /api/recommend
    ▼
[FastAPI 라우터]
    │  validate()
    ▼
[Phase 1] classify_metals()  ←  heavy_metals.json
    │
    ▼

## 13. 한계 및 고지사항

### 13.1 서비스 화면 필수 고지 문구

> 본 서비스의 모든 흡착 성능 추정치는 실험실 배치(batch) 조건의 문헌값에
> 배치→컬럼 보정(×0.40)을 적용한 **이론적 추정값**입니다.
> 실제 현장 성능은 유속, 온도, 공존 물질, 장기 파울링 등에 따라
> **이론값의 50–80% 수준**으로 나타날 수 있습니다.
> 본 서비스는 정보 제공 목적이며, 실제 수처리 시설 설계 전
> 반드시 공인 기관의 파일럿 실험 및 전문가 검토를 받으시기 바랍니다.

### 13.2 현재 모델의 주요 제약

| 제약 | 내용 |
|------|------|
| 경쟁 흡착 단순화 | 금속 조합별 선택성 차이 미반영 |
| 온도 효과 미반영 | 모든 데이터 ~25°C 기준 |
| 장기 파울링 미모델 | 운전 시간 따른 성능 저하 무시 |
| 수리학적 단순화 | HLR·EBCT 영향 비선형 특성 미반영 |
| Hg·Ni 데이터 부족 | 각 2건으로 신뢰도 낮음 |

### 13.3 참고문헌 접근

소재 상세 페이지와 레이어 상세 패널에 인용 출처(`reference_id`)를 표시한다.
전체 참고문헌 목록은 `db/README.md` 및 `/about` 페이지에서 확인 가능하도록 한다.
원문 논문 접근은 PubMed(PMC), ScienceDirect 외부 링크로 연결한다.
